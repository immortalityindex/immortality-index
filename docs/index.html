What This Is

A verifiable roadmap, not a prediction

The Immortality Index treats biological aging as an engineering problem. Rather than asking "when will we defeat aging?", it asks: what specific technical milestones need to happen, and which ones already have? Each of the 23 obstacles has 4 binary milestones — concrete, falsifiable achievements that peer-reviewed literature either supports or doesn't.

Progress is assessed daily by an AI system that scans current scientific literature. This isn't expert consensus — it's probabilistic, and may contain errors. The admin panel allows manual overrides when the AI gets it wrong.

How It Works

From research to dashboard

1

Milestone definitions

Each of the 23 obstacles has a Theoretical Target — the gold standard that fully solves that problem — plus 4 incremental milestones that form a verifiable progression toward it. Milestones are written to be binary: either the scientific evidence exists, or it doesn't.

2

Nightly AI literature scan

A Gemini 2.5 Flash model receives each obstacle's milestones plus a curated list of real-world scientific anchors (labs, papers, technologies). It searches its knowledge of the literature and returns a JSON assessment: completed or not, with brief evidence citations.

3

Firestore update & live display

Results are written to Firebase Firestore. The dashboard at immortality-index.pages.dev reads these in real time via Firestore's onSnapshot listener — no page refresh needed. Percentages update the moment any milestone changes.

4

Community corrections

The AI can be wrong. If a milestone is incorrectly marked or missed, anyone can file a GitHub Issue using the templates in this repo. Corrections are reviewed and applied manually via the admin panel.

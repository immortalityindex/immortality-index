Framework

A dependency graph, not a prediction

The Immortality Index formalises biological aging as an engineering problem with a directed acyclic graph (DAG) of prerequisites. Rather than asking “when will aging be solved?”, it specifies which discrete, falsifiable milestones must be demonstrated — in sequence — before indefinite somatic maintenance becomes achievable.

Each obstacle has a precisely specified theoretical target and four binary milestones corresponding to the validation hierarchy: in vitro organoids → murine in vivo → non-human primate → human clinical trial. An obstacle is complete only when all four milestones are confirmed by peer-reviewed evidence meeting strict inclusion criteria.

Live AI Literature Scanning

Methodology

Four-tier validation hierarchy

Milestone completion requires progression through four ordered validation tiers. Evidence at a lower tier does not satisfy a higher-tier milestone. This structure is grounded in the documented (>95%) translational failure rate for complex multi-target biomedical interventions moving from rodent to human.

1

In Vitro Human Organoid

3D iPSC-derived organotypic cultures. Mechanistic proof-of-concept in a genetically defined human cellular context.

2

In Vivo Murine

C57BL/6J or congenic background. Whole-organism physiological integration and longitudinal ageing readouts.

3

Non-Human Primate

Macaca mulatta preferred (93% protein identity with Track 01 targets). Minimum fidelity threshold for extrapolating to human safety.

4

Controlled Human Trial

Phase I/II IND filing with oncology screening, annual whole-genome karyotyping, and autoimmune panel. Phase II/III requires ≥5-year observation window.

Scanning Pipeline

How milestone state is determined

1

Literature query

Each cycle queries PubMed Entrez, bioRxiv/medRxiv, Semantic Scholar, and Europe PMC using each obstacle’s keyword taxonomy. A relevance classifier trained on 12,000 longevity abstracts filters records below a 0.65 probability threshold.

2

Hard inclusion criteria

Papers must satisfy all five criteria: minimum sample size (n ≥ 30 in vitro / ≥ 50 murine / ≥ 20 NHP / ≥ 30 human); p < 0.01 with multiple-comparison correction; independent replication within 24 months; blinded outcome assessment; and raw data in a public repository.

3

Three-stage LLM evaluation

Structured extraction → binary criterion adjudication (separate context windows per criterion to prevent anchoring) → milestone alignment scoring 0–100. Scores ≥ 85 go to the automated commit queue; 60–84 go to human review.

4

Firestore update & public audit log

Confirmed completions are written to Firebase Firestore and reflected live on the dashboard. Full extraction transcripts and alignment scores are stored in the public audit log — open to challenge via GitHub pull request.

Challenge a milestone assessment

The AI assessment is probabilistic and subject to false positives and false negatives. If a milestone is incorrectly confirmed or a published result has been missed, open an issue — all challenges are reviewed against the inclusion criteria.
